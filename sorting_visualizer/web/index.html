Project Summary

This project visualizes the internal mechanics of three fundamental sorting algorithms. Users can tune input size and animation speed to study how algorithmic complexity affects runtime and the number of operations performed.

  • Bubble Sort demonstrates quadratic behavior through repeated swaps.
  • Quick Sort highlights divide-and-conquer partitioning.
  • Merge Sort emphasizes stable merging and recursion depth.

Algorithmic Complexity

Bubble Sort

Best: O(n) — already sorted input.

Average/Worst: O(n^2) comparisons and swaps.

Quick Sort

Average: O(n log n) with good pivot selection.

Worst: O(n^2) when partitions are highly unbalanced.

Merge Sort

Always O(n log n) comparisons.

Requires O(n) extra memory for merging.
